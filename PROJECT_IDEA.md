# PROJECT_IDEA.md — ZenPulse: AI Meditation App

## Проблема
Пользователи тратят 15–30 минут на выбор медитации из сотен вариантов, не зная с чего начать. Статические библиотеки медитаций не учитывают текущее настроение пользователя — человек в стрессе получает тот же контент, что и человек в спокойном состоянии.

## Решение
Пошаговый пользовательский путь:
1. Открытие приложения → экран Paywall (демонстрация ценности Premium)
2. Активация пробного доступа → главный экран медитаций
3. Выбор настроения (3 эмодзи) → AI генерирует персональную аффирмацию
4. Выбор медитационной сессии из карточек (доступные vs заблокированные)

## Почему сейчас
- Expo SDK 54 + React Native 0.81 — стабильная платформа для быстрого прототипирования
- AI-генерация контента стала нормой — пользователи ожидают персонализации
- Рынок wellbeing-приложений: $4.2B в 2023, рост 8.9%/год

## Аудитория
**Основная**: 25–40 лет, работающие люди, испытывающие стресс, iPhone
**Вторичная**: HR-менеджеры, которые рекомендуют приложения сотрудникам

## Архитектура (MVP)
```
┌─────────────┐     ┌──────────────────┐     ┌───────────────────┐
│   Paywall   │────▶│  Home/Meditations │────▶│  AI Mood Feature  │
│  (экран 1)  │     │    (экран 2)      │     │  (внутри экрана 2)│
└─────────────┘     └──────────────────┘     └───────────────────┘
        ▲                    │
        └────────────────────┘
         (заблокированный контент → Paywall)
```

**Стек:**
- React Native + Expo SDK 54 (newArchEnabled: false)
- expo-router (навигация)
- expo-linear-gradient (premium UI)
- React Context (состояние isSubscribed)
- Mock LLM responses (AI аффирмации по настроению)
- StyleSheet + Dimensions API (адаптивная верстка)

**Почему этот стек:**
- Expo SDK 54 = Expo Go на iPhone без нативной сборки
- expo-router = файловая навигация, нет лишних импортов
- Mock LLM = нет зависимости от API-ключа, но выглядит реалистично
- Нет NativeWind/Tamagui = нет проблем с совместимостью на SDK 54
